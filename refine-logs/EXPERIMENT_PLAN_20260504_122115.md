# Experiment Plan

**项目**: DOPPLER  
**日期**: 2026-05-04  
**目标**: 用 claim-driven 实验验证 OR-guided RL 传感器布局是否提升 CDSTE 全网状态估计。

## Claim Matrix

| Claim | Evidence | Required? |
|---|---|---|
| C1: DOPPLER improves estimation accuracy | MAE/RMSE/MAPE/CRPS vs baselines across budgets/datasets | 必须 |
| C2: OR guidance is necessary | no-dual/no-MIP/no-local-search ablations | 必须 |
| C3: RL amortizes search | quality vs estimator-call curve; transfer across budgets | 必须 |
| C4: Layout is interpretable | budget shadow price, stability, selected-node diagnostics | 必须 |
| C5: Robustness | day split, seed, missing rate, sensor failure sensitivity | 强烈建议 |

## Datasets

1. PeMS7_228: first pilot and main anchor。
2. PeMS7_1026: scalability / transfer。
3. Seattle: external validation with different spatial matrix construction。
4. Optional SUMO Chengdu: only after main claims pass。

## Baselines

- Random layout, 10 seeds。
- Degree / closeness / betweenness centrality。
- k-medoids / spatial coverage。
- Historical variance / flow-volume top-k。
- Correlation coverage greedy。
- Viti-style null-space / observability metric。
- Owais-style sensitivity ranking if estimator gradient/proxy is available。
- Greedy forward selection under CDSTE proxy。
- Swap/local search。
- BO-lite / Embedding-BO-lite if implementation budget permits。
- DOPPLER variants。

## Main Experimental Blocks

### Block A: Evaluation Harness

Goal: 给定 sensor set，稳定复现 CDSTE 误差。

Runs:

1. PeMS7_228, budget 20%, random/centrality layouts, short training。
2. Fixed mask protocol sanity check：train/val/test hidden nodes consistent。
3. Full metrics JSON + manifest output。

Decision gate: 同一 layout 同一 seed 可复现，random baseline 方差合理。

### Block B: OR Surrogate Baselines

Goal: 证明 OR surrogate 本身是强 baseline，也能产出 guidance。

Runs:

1. AMPL coverage MIP at budgets 10/20/30%。
2. Correlation/facility-location MIP。
3. Null-space / observability heuristic。
4. LP relaxation + dual/reduced-cost extraction。

Decision gate: 至少一个 OR baseline 明显优于 random/topology-only。

### Block C: DOPPLER Training

Goal: 训练 add/swap policy。

Runs:

1. Behavior cloning from greedy/local-search/AMPL trajectories。
2. Offline RL reranking with CDSTE validation reward。
3. Potential-based reward shaping from OR surrogate score。

Decision gate: DOPPLER 在 held-out budget 上优于 OR baseline 或使用更少 CDSTE calls。

### Block D: Main Comparison

Goal: 支撑主 claim。

Grid:

- Datasets: PeMS7_228, PeMS7_1026, Seattle。
- Budgets: 5%, 10%, 20%, 30%, 40%。
- Missing rates: align with CDSTE `missing_rate` but treat budget as observed sensor ratio。
- Seeds: at least 5 for selection and 3 for CDSTE training。

Report:

- Mean/std MAE, RMSE, MAPE, CRPS。
- Paired tests or bootstrap confidence intervals。
- Estimator-call count and wall-clock。

### Block E: Ablation and Necessity

Variants:

- DOPPLER full。
- no-dual features。
- no-OR surrogate reward。
- no-local-search trajectories。
- behavior cloning only。
- RL from scratch。
- OR local search only。

Success: full model wins or clear evidence shows which component carries contribution。

### Block F: TS-Oriented Insight

Outputs:

- Budget marginal value curve。
- Sensor layout maps/topological distribution。
- Shadow price vs selected node interpretation。
- Robustness to sensor failure/dropout。
- Deployment recommendation: when additional sensors stop being cost-effective。

## First Three Runs

1. Build `layout_eval.py`: PeMS7_228, fixed sensor set, short CDSTE evaluation, JSON metrics。
2. Build AMPL coverage/correlation MIP and centrality baselines for PeMS7_228。
3. Run 20 random layouts + 5 deterministic baselines at budget 20% to estimate variance and effect size。

## Pilot Budget

No full GPU pilot was launched in this discovery pass because the current CDSTE path is DDP/CUDA-oriented and needs a small reproducible harness first. The planned pilot should stay under 2 GPU-hours per idea:

- `train_epochs <= 20`
- `diff_samples <= 4`
- PeMS7_228 only
- one seed for smoke test, then three seeds after harness validation

## Stop / Pivot Criteria

- If OR baselines do not beat random on PeMS7_228, revisit objective/mask protocol before RL。
- If DOPPLER cannot beat local search, reposition as amortized solver only if transfer/search-cost evidence is strong。
- If CDSTE variance exceeds expected method differences, freeze estimator or increase seeds before claiming method superiority。
