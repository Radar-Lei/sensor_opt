## Executive Summary

推荐不要把下一阶段理解为“重新找 idea”，而是把 TRACE-SL 重写为一个 **transparent reconstruction-aware sensor placement** 问题：给定历史全网状态、交通网络和预算，选择传感器集合，使透明 GLS/MAP 重构器在未观测节点上的重构风险最小，同时用 posterior trace / logdet / condition number 给出可解释的布局质量 certificate。

最稳妥的投稿路线是 **Transportation Science 第一版**：主贡献为 formal formulation + posterior-error 理论桥梁 + RCSS portfolio/candidate-space search + 跨网络/鲁棒实验。TR Part B 可以作为增强版，但必须再加入更硬的数学性质、复杂度/局部最优分析，以及更一般的 stochastic/bilevel sensor placement 表述。

## Current Evidence Snapshot

已有证据已经足够支撑“方法值得打磨”，但不支持“单一 validation-swap RCSS 在所有预算都是最强”的过强叙事。

- PeMS7_228 Stage 11 10-split：validation-swap RCSS 相比 best-random by validation 在 10/20/30% budget 分别降低 MAE 0.0858 / 0.0874 / 0.1167；Wilcoxon p 分别为 0.0273 / 0.0059 / 0.0020。
- 但 PeMS7_228 10% budget 下 `multistart_swap_by_validation` mean MAE 为 3.5783，优于 `validation_swap_selected` 的 3.6055；因此主 claim 必须承认 low-budget multistart 是强特例。
- PeMS7_1026 5-split：validation-swap RCSS 对 best-random by validation 全预算 5/5 wins，delta 为 -0.0879 / -0.2105 / -0.2531。
- Seattle outputs 已在仓库中存在：`TRC-23-02333/trace_sl_results/seattle_stage11_auto_weight_light/`，可作为非 PeMS 轻量外部验证，但最好在论文中明确其 light protocol。
- GLS/MAP certificates 与 hidden MAE 稳定相关：PeMS7_228 posterior trace Spearman 约 0.851，PeMS7_1026 约 0.931，Seattle 约 0.874；这为理论桥梁提供了实证支撑。

## Literature / Positioning Landscape

TRACE-SL 应主动与四类文献区分：

1. **经典交通 sensor location / observability**：多关注 deterministic full observability 或 link-flow inference；TRACE-SL 关注有限预算 partial observation 下的 reconstruction quality 和 uncertainty。
2. **A/D-optimal experimental design / Kalman sensor placement**：提供 posterior covariance surrogate；TRACE-SL 的新意是把这些 certificates 与真实 validation reconstruction error、candidate portfolio 和交通网络部署协议结合。
3. **GSP / graph sampling / compressed sensing**：提供 sparse graph signal reconstruction 基线；TRACE-SL 不声称发明重构器，而是优化“哪个传感器集合让透明重构器可靠”。
4. **learning-based traffic imputation / GNN**：可作为外部重构器 sanity check，但不能成为主方法；主线必须保持透明 inverse model + OR search。

## Ranked Journal-Upgrade Routes

### 1. RECOMMENDED — RCSS Portfolio as Validation-Calibrated Stochastic Sensor Design

**Core claim**  
TRACE-SL constructs a transparent, reconstruction-aware candidate space in which validation-calibrated portfolio selection chooses among certificate-guided, multistart, coverage, scenario-risk, and swap-refined layouts; this consistently improves hidden-link GLS/MAP reconstruction over random, validation-selected random, variance/topology baselines, and standard OR baselines, while exposing when low-budget multistart refinement is preferable.

**新增理论**
- Formal objective:  
  `min_{S: |S|<=k} E_{t in test}[ell(x_hat_{H,t}(S), x_{H,t})]`, where `x_hat` is GLS/MAP and `H=V\S`.
- Tractable surrogate: validation MAE + posterior trace/logdet/condition + scenario CVaR + coverage.
- Linear-Gaussian bridge: posterior covariance trace equals expected squared hidden reconstruction error under the model; Gaussian absolute-error approximation links marginal variance to expected MAE.
- Portfolio definition fixed before testing: candidate generators are predeclared, validation selects one layout or score vector; no post-hoc cherry-picking.

**新增代码/实验**
- Rename and implement the final selector as a pre-specified `TRACE-SL portfolio selector`, not “validation-swap is always final”.
- Extend PeMS7_1026 to 10 splits if time permits; otherwise report five-split as external validation with exact win counts.
- Add/clean baselines: A-trace, D-logdet, GSP sampling, QR/SVD/POD, observability-inspired coverage/cut baseline, simple learning-based reconstructor evaluation.
- Add robustness suite: sensor failure, observation noise, missingness, nonuniform cost, temporal drift, candidate-count sensitivity.
- Treat Seattle as light heterogeneous validation; either include its outputs in the result manifest or label it secondary.

**预计工作量**: 2-3 周可形成 TS first draft；TRB 需要再加 2-4 周理论。  
**最大风险**: 如果 portfolio selector 被写成“把所有赢家放进候选池然后验证挑”，审稿人会质疑 overfitting；必须固定 selection protocol 和 validation/test separation。  
**TS fit**: 很强。  
**TRB fit**: 中等；需要更多定理与算法性质。

### 2. Theory-First Posterior Certificate Paper

**Core claim**  
For transparent GLS/MAP traffic reconstruction, posterior covariance certificates are principled predictors of hidden-link reconstruction risk; sensor placement can therefore be optimized through A/D-optimal and risk-sensitive covariance objectives.

**新增理论**
- 严格证明 posterior trace / logdet 与 expected reconstruction error 的关系。
- 给出 condition number 对稳定性的 perturbation bound。
- 对 greedy A-trace 或 logdet 子问题讨论 monotonicity / approximate submodularity 条件。

**新增代码/实验**
- 以 A/D-optimal、scenario CVaR、condition-aware variants 为核心 baseline。
- 重点做 certificate-vs-error scatter、rank correlation、out-of-network transfer。
- RCSS 只作为 practical heuristic，不作为主角。

**预计工作量**: 3-5 周。  
**最大风险**: 贡献可能被认为是把已知 Gaussian conditioning / experimental design 搬到 PeMS，工程新意不足。  
**TS fit**: 中强。  
**TRB fit**: 中强，前提是证明写得硬。

### 3. Robust Deployment Stress-Test Paper

**Core claim**  
Reconstruction-aware sensor layouts are valuable because they remain effective under deployment disturbances: sensor failures, noise, missingness, budget/cost heterogeneity, and temporal drift.

**新增理论**
- 将 scenario CVaR / robust objective 形式化。
- 给出 robustness protocol 与 distribution-shift interpretation。

**新增代码/实验**
- 系统加入 failure 5/10/20%、noise levels、missingness blocks、nonuniform cost knapsack、month/week temporal splits。
- 报告 degradation curves 和 layout stability。

**预计工作量**: 2-4 周，主要花在实验矩阵和图表。  
**最大风险**: 如果缺少 formal formulation，会像“加了一堆 stress tests”的实验论文，方法学贡献不够集中。  
**TS fit**: 强，尤其适合 planning/deployment insight。  
**TRB fit**: 中等偏弱，除非 robust optimization 写得严谨。

### 4. Baseline-Centric Benchmarking / Replication Paper

**Core claim**  
A fair benchmark of traffic sensor placement for transparent full-network reconstruction shows when classic observability, experimental design, graph sampling, random validation, and TRACE-SL portfolio search succeed or fail.

**新增理论**
- 主要是 taxonomy 和统一 evaluation protocol。

**新增代码/实验**
- 补齐 TSLP-inspired、GSP sampling、QR/POD、A/D-optimal、learning-based reconstruction baselines。
- 多网络、多预算、统一统计。

**预计工作量**: 2-3 周。  
**最大风险**: 容易被认为是 benchmark 而非 Transportation Science 方法论文；如果 TRACE-SL 优势不稳定，主贡献会变弱。  
**TS fit**: 中等。  
**TRB fit**: 弱。

### 5. TR Part B Ambitious Bilevel / Stochastic Optimization Version

**Core claim**  
Traffic sensor placement is a stochastic bilevel design problem where the upper level selects sensors under budget/cost/failure scenarios and the lower level solves transparent MAP reconstruction; RCSS is a scalable local-search/portfolio approximation with certificate-based risk control.

**新增理论**
- Bilevel/stochastic formulation。
- Posterior-risk theorem。
- Algorithmic complexity and local optimality condition for validation-aware swap。
- Optional submodular-like or approximation discussion for restricted covariance objectives。

**新增代码/实验**
- Same as Route 1 plus stronger cost/failure/drift settings and runtime scaling。

**预计工作量**: 5-8 周。  
**最大风险**: 时间成本高；如果理论不能真正超出标准 Gaussian conditioning，会显得包装过重。  
**TS fit**: 强但可能过度。  
**TRB fit**: 最强，但风险最高。

## Recommended Route

选择 **Route 1: RCSS Portfolio as Validation-Calibrated Stochastic Sensor Design**。

原因：它最忠实于当前结果，也最能化解关键弱点。10% budget 下 multistart 更强不应被隐藏，而应变成方法定义的一部分：TRACE-SL 的贡献不是单一 heuristic 永远赢，而是建立一个透明、certificate-guided、validation-calibrated 的布局候选空间，并通过严格的 train/validation/test protocol 选择适合预算和网络的布局。这个叙事对 Transportation Science 友好，也为 TR Part B 预留理论增强空间。

Recommended claim wording:

> TRACE-SL is a transparent reconstruction-aware sensor placement framework. It builds a pre-specified portfolio of certificate-guided, scenario-risk, coverage-diverse, random-validation, and validation-swap layout generators, then selects layouts through nested validation for a GLS/MAP reconstruction objective. Across PeMS and Seattle networks, the resulting layouts reduce hidden-link reconstruction error relative to random, validation-selected random, variance/topology, and standard OR baselines; at very low budgets, multistart validation refinement emerges as a strong portfolio component rather than a failure of the framework.

Avoid:

- “Certified” unless qualified as empirical/posterior certificates.
- “Best at all budgets.”
- Presenting auto-weight or final candidate choice as manually tuned after seeing test results.
- Treating Seattle as full-strength external validation unless its protocol/result directory is fully curated in the repository and paper artifact list.
