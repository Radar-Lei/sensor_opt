<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| validator-to-filesystem | Validator reads generated artifacts and curated aggregate outputs as untrusted local files. |
| validator-to-subprocess | Validator runs local validators and dry-run launcher smoke commands. |
| docs-to-paper | README/audit content becomes manuscript-facing guidance and must not point at missing or raw-data evidence. |
| raw dataset boundary | Local raw datasets remain outside committed paper-visible artifacts. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-07 | Tampering | final validator schema checks | mitigate | Fail closed on missing files, empty CSVs, missing provenance columns, missing method/layout rows, and nonzero subprocess exits. |
| T-06-08 | Information Disclosure | docs and raw-data hygiene | mitigate | Scan tracked raw dataset paths and paper-visible docs for raw dataset evidence citations. |
| T-06-09 | Denial of Service | launcher smoke checks | mitigate | Use `DRY_RUN=1`, `bash -n`, and small explicit env overrides; do not run long experiments. |
| T-06-10 | Repudiation | REPRO audit and validator output | mitigate | Emit requirement-specific PASS/FAIL rows and map every REPRO requirement to artifacts and commands in the audit. |
| T-06-SC | Tampering | package installs | accept | No package-manager install task is planned; validation uses existing local Python dependencies only. |
</threat_model>
